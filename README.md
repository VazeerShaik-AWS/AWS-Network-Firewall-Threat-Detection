# Threat Detection and Traffic Inspection using AWS Network Firewall

## Overview

Implemented advanced network security controls using AWS Network Firewall to inspect, monitor and filter VPC traffic, enabling centralized threat detection and protection.

## Objective

* Inspect and control network traffic at VPC level
* Detect and block malicious traffic patterns
* Centralize network security enforcement
* Improve visibility into network-level threats

## Problem Statement

* Security Groups cannot perform deep packet inspection
* Limited visibility into network traffic and threats
* Lack of centralized control over network security policies

## Architecture

* AWS Network Firewall deployed within the VPC for traffic inspection
* Configured both stateless and stateful firewall rules
* Routed traffic through firewall endpoints for inspection
* Integrated logging with Amazon CloudWatch for monitoring and analysis

## AWS Services Used

* AWS Network Firewall
* Amazon CloudWatch Logs

## Outcome / Business Impact

* Strengthened network security posture with deep packet inspection
* Improved visibility into traffic patterns and potential threats
* Reduced attack surface by filtering malicious traffic
* Enabled centralized and scalable security enforcement
